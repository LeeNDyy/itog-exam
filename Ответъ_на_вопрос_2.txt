Особенности проектирования сервисов с протоколом REST

Протокол REST — как я это понял, это набор правил для взаимодействия между клиентом и сервером по HTTP.
В REST используются стандартные HTTP-методы:

  GET — получить данные
  POST — создать новые данные
  PUT — обновить данные
  DELETE — удалить данные
  Там еще есть пару методов, но эти 4 основные, которые постоянно используются
Параметры запроса передаются через URL или тело запроса, чаще в формате JSON.
    Ответы сервера обычно приходят в формате JSON, это удобно приложению, то есть ему попросту легче обработать жэти данные

Плюсы протокола REST

Простота и удобство — легко понять и использовать.
Использование стандартных HTTP методов.
Хорошая масштабируемость.
Независимость от платформы и языка программирования.
Легко работать с кэшированием, что повышает производительность.

Минусы протокола REST
Особо прям минусов нет, но есть пару, такие как:
    1. Иногда сложно описать сложные зависимости между ресурсами.
    2. Нет сохранения состояния, то есть сервер не помнит клиента между запросами нужно постоянно передавать всю важную информацию
